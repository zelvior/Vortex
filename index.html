Will my data be lost if I clear my browser cache?

Yes — if you clear localStorage, local data is lost. Always Export a backup before clearing. Use Cloud Sync (Supabase or Firebase) for persistent backup.

Can I use this on multiple devices?

Yes — connect Supabase or Firebase. in Settings → Cloud Sync, then use Push/Pull or enable Auto Sync. Data is synced through your own cloud account.

Is there a mobile app?

No native app — but you can install Vortex as a PWA. On Android: Chrome menu → "Add to Home Screen". On iOS: Safari → Share → "Add to Home Screen". It runs fullscreen and works offline.

Does Vortex send any data to Zelvior or any server?

No. Zero. Nothing is sent anywhere unless you explicitly connect Supabase or Firebase. — and even then, data goes to your own account, not ours.

What's the storage limit?

localStorage is typically 5–10 MB per browser. IndexedDB (used as a secondary store) can hold much more. For most users with hundreds of entries and no cover images stored locally, this is more than enough.

Poster fetch says "No results" — what do I do?

Try a shorter or different title spelling. Some titles have region-specific names. You can also paste a direct image URL from Google Images or any website.

Help & Tutorial

Complete Connection Guide

This guide covers both Supabase (SQL-based, great for structured data) and Firebase Realtime Database (JSON-based, zero setup, instant writes). Pick whichever suits you — or connect both. No coding experience needed.

Jump to Firebase Guide ↓ Support Dev — Free!
1

Step One

What is Supabase?

Supabase is a free, open-source cloud database platform — think of it as a simple online storage system for your data.

When you connect Supabase to Vortex, your media entries get saved to the cloud — so you can access them from any device, anytime. Your data stays in your own Supabase account. Vortex never touches it.

Supabase's free tier gives you 500 MB storage and 2 projects — more than enough for Vortex.
2

Step Two

Create a Supabase Project

Follow these steps to create your free project:

  1. ① Go to supabase.com and click "Start your project". Sign in with GitHub or email.
  2. ② Click "New Project". Choose an organization (or create one), then give your project a name — e.g. mediavault.
  3. ③ Set a strong database password (save it somewhere safe — you won't need it in this app, but it's important).
  4. ④ Choose a region close to you, then click "Create new project". Wait about 1 minute for it to finish setting up.
3

Step Three

Find Your API URL & Key

Once your project is ready:

  1. ① In the left sidebar, click the ⚙ Settings gear icon, then go to API.
  2. ② Under "Project URL", copy the URL — it looks like https://abcxyz.supabase.co.
  3. ③ Under "Project API keys", copy the Publishable key (new format: sb_publishable_...) or the anon / public key (old format: eyJ...). Both work.
Use the Publishable key (sb_publishable_...) or anon / public key (eyJ...) — never the secret or service_role key. Those have full admin access and must never be exposed in a browser.
4

Step Four

Connect to Vortex

  1. ① Go to the Settings tab in this app and scroll to the Supabase Connection section.
  2. ② Paste your Project URL into the URL field and your anon key into the API Key field.
  3. ③ Click Connect. The app will validate your credentials by making a test request to your project.
  4. ④ If successful, the status badge turns green. Your credentials are saved only in your browser — never sent anywhere else.
?

Troubleshooting

Common Errors & Fixes

"Invalid URL format"

Make sure the URL starts with https:// and ends with .supabase.co. No trailing slash.

"Connection failed" / Network error

Check your internet connection. Also verify the URL is correct — even a small typo will cause this.

"Invalid API key" / 401 Unauthorized

You may have used the secret / service_role key. Go back to Supabase → Settings → API and copy the Publishable key (sb_publishable_...) or the anon / public key (eyJ...).

Status stays "Not connected" after clicking Connect

Make sure both fields are filled. If using a browser extension that blocks requests (like uBlock Origin), try temporarily disabling it for this page.

Still stuck?

Reach out to Zelvior via email at Zelvior@proton.me or check the YouTube channel for a video walkthrough.

Advanced

Push vs Pull — What's the difference?

Push to Cloud

Sends your local vault data up to Supabase. Use this after making changes locally that you want backed up.

Pull from Cloud

Downloads your data from Supabase and replaces your local vault. Use this to restore or sync from another device.

Both operations overwrite the destination. Pull will replace all local data. Push will replace all cloud data. There is no merge — always export a backup first if unsure.
7

Advanced

How Auto-Sync Works

When Auto Sync is enabled in Settings → Supabase Advanced, the app automatically pushes your vault to Supabase at the interval you set (30 seconds to 10 minutes).

  1. ① Enable Auto Sync in Settings.
  2. ② Choose your sync interval. 1 minute is recommended for most users.
  3. ③ Every sync is logged in the Sync Log visible in Settings — green dot = success, red = failed.
  4. ④ If a sync fails, you'll get a notification. Check your connection or re-test in Settings.

Auto Sync only runs while the app is open in your browser tab. Closing the tab stops the sync timer.

8

Advanced

Setting Up the Supabase Table

Before you can push data, you need to create the table in your Supabase project. The easiest way is to use the built-in SQL generator:

  1. ① In Settings → Supabase Advanced, click View SQL Schema.
  2. ② Copy the SQL and click Open Supabase to go to your project.
  3. ③ In the Supabase dashboard, go to SQL Editor → New Query.
  4. ④ Paste the SQL and click Run. Your table is ready.
9

Security

Row Level Security (RLS)

By default, Supabase tables have Row Level Security disabled — meaning anyone with your anon key can read/write. For personal use with a private key this is acceptable, but you can enable RLS for extra safety.

Simple RLS Policy (allow all with anon key)

ALTER TABLE mediavault ENABLE ROW LEVEL SECURITY; CREATE POLICY "allow_all" ON mediavault FOR ALL USING (true) WITH CHECK (true);

Run this in Supabase → SQL Editor after enabling RLS in your table's Auth settings. This allows all reads and writes with the anon key — suitable for single-user personal vaults.

Firebase Guide — Step 1

What is Firebase Realtime Database?

Firebase Realtime Database is Google's cloud-hosted JSON database. Unlike Supabase (which uses SQL tables), Firebase stores everything as a giant JSON tree. There are no tables, no schemas, no SQL — you just write to a path and the data appears instantly.

✓ Advantages

  • • Free tier: 1 GB storage, 10 GB/month transfer
  • • No table setup required at all
  • • Works directly from any browser
  • • Real-time updates across devices

ℹ️ Good to know

  • • Test mode = no auth needed (easy start)
  • • Locked mode = needs API key
  • • Data is JSON, not SQL rows
  • • Free plan is generous for personal use
2

Firebase Guide — Step 2

Create a Firebase Project

  1. ① Go to console.firebase.google.com and sign in with your Google account.
  2. ② Click Add project. Give it a name (e.g. mediavault). You can disable Google Analytics if you don't need it.
  3. ③ Once created, in the left sidebar go to Build → Realtime Database.
  4. ④ Click Create Database. Choose a location. When asked about security rules, select "Start in test mode" — this lets you connect without any API key.
  5. ⑤ Click Enable. Your database is ready in seconds.
Test mode expires after 30 days. Before expiry, go to Rules tab and set ".read": true, ".write": true to keep it open. For production, add proper security rules.
3

Firebase Guide — Step 3

Get Your Database URL & API Key

Database URL (required):

  1. ① In the Firebase console, go to Build → Realtime Database.
  2. ② At the top of the Data tab you'll see your URL — it looks like: https://your-project-default-rtdb.firebaseio.com
  3. ③ Copy that full URL and paste it into the Database URL field in Settings.

Web API Key (only needed if your rules require auth):

  1. ① In the Firebase console, click the ⚙ gear icon → Project Settings.
  2. ② Under the General tab, scroll down to find Web API Key — it starts with AIzaSy...
  3. ③ If you're using Test mode, you can leave the API Key field blank — Vortex will connect without it.
4

Firebase Guide — Step 4

Connect & Use

  1. ① Go to Settings and scroll to the Firebase Realtime Database section.
  2. ② Paste your Database URL. If your rules need auth, also paste the Web API Key.
  3. ③ Click Connect Firebase. The app writes a test entry and reads it back to verify. No manual setup needed.
  4. ④ Once connected (status turns green), use Push to Firebase to upload your vault, or Pull from Firebase to download it.
  5. ⑤ Enable Auto Sync to automatically back up your vault on a schedule.
🔒

Firebase Guide — Security

Firebase Security Rules

In the Firebase console → Realtime Database → Rules tab, you control who can read/write.

Test mode (easiest — anyone can read/write):

{ "rules": { ".read": true, ".write": true } }

Secure mode (locked — no access without auth):

{ "rules": { ".read": false, ".write": false } }

For personal single-user use, Test mode is fine — just don't store sensitive personal information. The URL is not publicly listed, so it's reasonably private.

Firebase Guide

Firebase Troubleshooting

"Permission denied" error

Your database rules are in Locked mode. Either switch to Test mode in the Rules tab, or provide your Web API Key in the settings.

"Invalid URL" error

Make sure the URL starts with https:// and ends with .firebaseio.com or .firebasedatabase.app.

Test mode expired

Firebase test mode expires after 30 days. Go to Rules tab and manually set ".read": true, ".write": true to re-enable open access.

Connection succeeds but Push/Pull fails

Check that the Path / Node field in settings matches a path your rules allow access to. Default is mediavault.
